# CLAUDE.md — 루프 기반 규격 (Loop Constitution)

> 계양부동산·데이터부동산연구소 | 담당자 홍익
> 버전 v1.0 | 2026-07-01
> 용도: 모든 Claude Code 프로젝트 루트에 `CLAUDE.md`로 드롭인. 루프의 '헌법'.
> 이 파일은 매 세션 자동 로드되어 에이전트의 기본 전제가 됩니다.

---

## 0. 이 파일의 역할

- 프로젝트의 **목표·경계·검증기준·종료조건**을 한 곳에 정의합니다.
- 공통 규칙(§1~§7)은 6개 도메인 전부 동일하게 유지합니다.
- 프로젝트마다 **§8 블록만 교체**합니다.
- 참조 상세 규칙은 프로젝트 지식파일: `개발환경_표준` / `코딩_규칙` / `배포_체크리스트` / `파일전달_규칙` / `보유자산_인벤토리`.

---

## 1. 작업 모드 (항상 모드 먼저 선언)

| 모드 | 허용 | 금지 |
|---|---|---|
| **대화 모드** | 논의·설계·옵션 제시 | 파일 쓰기, API 호출, 배포 |
| **실행 모드** | "적용해줘" 이후 쓰기·API·배포 | 확인 없는 광범위 변경 |

- 모드가 애매하면 실행 전 **1회 확인**합니다.
- 의사결정은 **번호 옵션 제시 → 순차 선택**. 한꺼번에 결정하지 않습니다.

---

## 2. 루프 프로토콜 (핵심)

단순 질의를 제외한 모든 작업은 아래 6단계 루프로 처리합니다.

1. **계획(Plan)** — 목표를 *검증 가능한 완료조건*으로 분해합니다.
2. **실행(Act)** — 가장 작은 일관된 변경 1개만 수행합니다.
3. **관찰(Observe)** — 테스트·로그·실행결과를 확인합니다. **관찰 없이 다음 단계 금지.**
4. **검증(Verify)** — §5 검증 게이트로 판정합니다.
5. **결정(Decide)** — 통과 → 다음 단위 / 실패 → 원인 분류 후 수정 / **같은 오류 2회 → 중단·보고**.
6. **종료(Stop)** — 완료조건 충족 또는 §4 종료조건 도달 시 정지하고 사람 검토를 요청합니다.

---

## 3. 제작자 / 검증자 분리

- **제작자(maker):** 만듭니다. 자기 작업을 스스로 '완료'로 선언하지 않습니다.
- **검증자(verifier):** 독립 기준으로 검사합니다. '완료' 판정은 **검증자만** 내립니다.
- **사람 최종 검토는 어떤 경우에도 유지합니다.** (루프가 좋아질수록 이해 못 한 산출물이 더 빨리 쌓이므로)

---

## 4. 종료조건 (아래 중 하나라도 해당하면 반드시 멈춤)

- 완료조건을 전부 충족
- 동일 오류 2회 반복(스피닝)
- 하드 블로커: 자격증명 부재·미정의 동작·범위 밖 판단
- 예산 근접: 토큰·시간·CODEF 일일 한도
- 민감정보 노출 또는 보안 위험 감지

---

## 5. 검증 게이트 (공통) — 하나라도 미충족이면 '미완료'

- [ ] 전체 파일 교체 (부분 스니펫 아님)
- [ ] API 키·환경변수 하드코딩 없음
- [ ] 민감 개인정보(주민번호·계좌 등) 미포함
- [ ] 디자인 컬러 표준 준수 `#1E3A8A` / `#3B82F6`
- [ ] 파일명 날짜 접미사 규칙 `_YYYYMMDD`
- [ ] 웹검색으로 가져온 수치는 핵심 1~2개 직접 확인
- [ ] 플랫폼 역할 경계 준수 (Waveon=리드 / Streamlit=정부API / Artifacts=팀공유)
- [ ] (도메인별 추가 기준 §8 통과)

---

## 6. 복리(Compounding) 메커니즘 — 가속의 핵심

> 목표: 품질이 은행 복리처럼 매 루프마다 누적·가속되도록 만든다.

- 루프 1회 종료 시마다 **'배운 것 1줄'**을 `./LEARNINGS.md`에 기록합니다.
- 같은 교훈이 **3회 누적**되면 이 파일의 규칙(§5 또는 §8)에 **반영**합니다. → 다음 루프는 더 높은 지점에서 출발합니다.
- **격주 1회 회고:** `LEARNINGS.md` 검토 → 검증기준·프롬프트·서브에이전트 업데이트.
- 원칙 2가지:
  1. **같은 실수는 두 번까지만.** 세 번째 전에 규칙화합니다.
  2. **좋은 패턴은 즉시 표준화.** 재사용 가능한 형태로 승격합니다.

**LEARNINGS.md 기록 형식**

```
## YYYY-MM-DD | [프로젝트] | [작업]
- 배운 것:
- 다음에 바꿀 것:
- 규칙 반영 여부: (미반영 / 1회 / 2회 / 3회→규칙화 완료)
```

## 7. 보안·경계 (무조건 최우선)

- 승인 정책: 읽기전용 자유 / 쓰기·API 확인 후 / **"유사 명령 다시 묻지 않기" 광범위 옵션 선택 금지**.
- 에이전트는 취약점을 **대량으로** 생산할 수 있습니다 → 배포 전 검증자 보안 항목을 필수 통과합니다.
- 파일 전달은 `present_files` 다운로드 → `mv` 우선. heredoc 붙여넣기는 지양합니다.
